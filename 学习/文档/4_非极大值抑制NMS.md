# 非极大值抑制 (NMS)

## 概述

非极大值抑制（Non-Maximum Suppression, NMS）是目标检测的**最后一步**——后处理算法。它的作用是从大量重复的检测框中，**只保留最好的那一个**。

> **类比**：考试改卷时，同一个学生的名字出现了 5 次（因为不同老师都登记了一次）。NMS 就是去重：只保留分数最高的那条记录，删掉其他重复的。

---

## 为什么需要 NMS

### 问题的本质

YOLO 有 3 个尺度的特征图，合计产生 **80×80 + 40×40 + 20×20 = 8,400** 个预测位置。每个位置都可能预测一个框。

对于画面中一个网球运动员，可能有**几十个框**都在描述同一个人：

```
                原始检测结果（一个人被检测了很多次）

          ┌─────────────────────┐  conf=0.92
          │ ┌───────────────────┼──┐  conf=0.88
          │ │ ┌─────────────────┼──┼──┐  conf=0.75
          │ │ │                 │  │  │
          │ │ │     人物        │  │  │
          │ │ │                 │  │  │
          │ │ └─────────────────┼──┼──┘
          │ └───────────────────┼──┘
          └─────────────────────┘

          这些框大部分重叠，只需要保留一个！
```

我们需要的结果是：**每个目标只保留一个最好的框**。

## NMS 算法——手把手走一遍

### 输入

假设 YOLO 对一张图输出了 5 个检测框：

| 编号 | 框坐标 (x1,y1,x2,y2) | 置信度 |
|------|----------------------|--------|
| A | (100, 100, 300, 400) | **0.92** |
| B | (110, 105, 310, 410) | 0.88 |
| C | (105, 98, 305, 405) | 0.75 |
| D | (500, 200, 650, 500) | 0.85 |
| E | (510, 210, 660, 510) | 0.72 |

其中 A、B、C 描述同一个人（框高度重叠），D、E 描述另一个人。

**IoU 阈值设为 0.5**

### 第 1 轮

**步骤 1**：按置信度排序 → A(0.92), B(0.88), D(0.85), C(0.75), E(0.72)

**步骤 2**：取出最高分 A(0.92) → **保留 A** ✓

**步骤 3**：计算 A 和剩余所有框的 IoU：

```
IoU(A, B) = 交集面积 / 并集面积 ≈ 0.85  > 0.5 → 删除 B ✗
IoU(A, D) ≈ 0.0                          < 0.5 → 保留 D
IoU(A, C) ≈ 0.80                         > 0.5 → 删除 C ✗
IoU(A, E) ≈ 0.0                          < 0.5 → 保留 E
```

**结果**：B 和 C 被抑制（因为和 A 高度重叠，被认为是同一个人的重复检测）

剩余：D(0.85), E(0.72)

### 第 2 轮

**步骤 2**：取出最高分 D(0.85) → **保留 D** ✓

**步骤 3**：计算 D 和 E 的 IoU：

```
IoU(D, E) ≈ 0.78  > 0.5 → 删除 E ✗
```

剩余：空

### 最终结果

```
保留: A(0.92) 和 D(0.85)  ← 每个人只剩一个框！
删除: B, C, E              ← 重复框被抑制
```

```
         NMS 之前                          NMS 之后

    ┌──┬──┬──┐     ┌──┬──┐           ┌─────┐      ┌─────┐
    │A │B │C │     │D │E │           │  A  │      │  D  │
    │  │  │  │     │  │  │     →     │     │      │     │
    └──┴──┴──┘     └──┴──┘           └─────┘      └─────┘
     3 个重叠框     2 个重叠框         1 个框        1 个框
```

## 完整 Python 实现

```python
import numpy as np

def calculate_iou(box1, box2):
    """计算两个框的 IoU, 格式 [x1, y1, x2, y2]"""
    x1 = max(box1[0], box2[0])
    y1 = max(box1[1], box2[1])
    x2 = min(box1[2], box2[2])
    y2 = min(box1[3], box2[3])
    
    inter = max(0, x2 - x1) * max(0, y2 - y1)
    area1 = (box1[2]-box1[0]) * (box1[3]-box1[1])
    area2 = (box2[2]-box2[0]) * (box2[3]-box2[1])
    union = area1 + area2 - inter
    
    return inter / union if union > 0 else 0

def nms(boxes, scores, iou_threshold=0.5):
    """
    boxes:  list of [x1, y1, x2, y2]
    scores: list of float (置信度)
    返回:   保留的框的索引
    """
    # 按置信度排序（从高到低）
    order = np.argsort(scores)[::-1]
    
    keep = []
    while len(order) > 0:
        # 取出当前最高分的框
        i = order[0]
        keep.append(i)
        
        # 计算这个框和剩余所有框的 IoU
        remaining = []
        for j in order[1:]:
            iou = calculate_iou(boxes[i], boxes[j])
            if iou < iou_threshold:
                remaining.append(j)   # IoU 低于阈值，保留
            # else: IoU >= 阈值，抑制（不加入 remaining）
        
        order = remaining
    
    return keep

# 使用示例
boxes = [
    [100, 100, 300, 400],  # A
    [110, 105, 310, 410],  # B
    [105, 98, 305, 405],   # C
    [500, 200, 650, 500],  # D
    [510, 210, 660, 510],  # E
]
scores = [0.92, 0.88, 0.75, 0.85, 0.72]

kept = nms(boxes, scores, iou_threshold=0.5)
print(f"保留的框: {kept}")         # [0, 3]  即 A 和 D
print(f"保留的分数: {[scores[i] for i in kept]}")  # [0.92, 0.85]
```

## IoU 阈值的影响

阈值越低 → 越严格（删得越多） → 可能漏检
阈值越高 → 越宽松（保留更多） → 可能有重复框

| 阈值 | 行为 | 适用场景 |
|------|------|----------|
| **0.3** | 非常严格，稍有重叠就删 | 目标稀疏、不可能重叠 |
| **0.45** | YOLOv8 默认值 | 一般场景 |
| **0.5** | 常用标准 | 大多数场景 |
| **0.7** | 宽松，大量重叠才删 | 密集目标（如人群） |

**网球场景**：画面中通常只有 1~2 个人，用 0.45 足够。

## NMS 的改进版本

### 问题 1：硬删除太暴力

标准 NMS 直接删除重叠框（硬删除）。但如果两个目标真的挨得很近呢？

```
两个人站很近，框有 60% 重叠:
┌────────────┐
│  人1  ┌────┼──────┐
│       │ 重叠│ 人2  │
└───────┼────┘      │
        └───────────┘

IoU = 0.6 > 0.5 → 人2 被错误删除！
```

### 解决方案 1：Soft-NMS

不直接删除，而是**降低重叠框的置信度**。重叠越多，置信度降得越多：

```python
def soft_nms(boxes, scores, sigma=0.5, score_threshold=0.01):
    """
    不删除框，而是降低重叠框的置信度
    """
    N = len(boxes)
    indices = list(range(N))
    
    keep = []
    while indices:
        # 找当前最高分
        max_idx = max(indices, key=lambda i: scores[i])
        keep.append(max_idx)
        indices.remove(max_idx)
        
        # 降低重叠框的置信度
        to_remove = []
        for j in indices:
            iou = calculate_iou(boxes[max_idx], boxes[j])
            # 高斯衰减：IoU 越大，衰减越多
            scores[j] *= np.exp(-iou**2 / sigma)
            
            if scores[j] < score_threshold:
                to_remove.append(j)  # 置信度太低才删除
        
        for j in to_remove:
            indices.remove(j)
    
    return keep
```

**对比**：

| 方法 | IoU=0.6 的框 | 效果 |
|------|-------------|------|
| 标准 NMS | 直接删除 | 可能漏检 |
| Soft-NMS | 置信度从 0.85 降到 0.59 | 保留但降权 |

### 解决方案 2：Matrix NMS

用矩阵运算并行计算所有框对之间的 IoU，效率更高：

```
IoU 矩阵 (5×5):
       A     B     C     D     E
A  [ 1.00  0.85  0.80  0.00  0.00 ]
B  [ 0.85  1.00  0.78  0.00  0.00 ]
C  [ 0.80  0.78  1.00  0.00  0.00 ]
D  [ 0.00  0.00  0.00  1.00  0.78 ]
E  [ 0.00  0.00  0.00  0.78  1.00 ]

→ 一次矩阵运算完成所有 IoU 计算，比逐对循环快得多
```

## NMS 在 YOLO 中的完整流程

```
YOLO 前向传播
    │
    ▼ 输出 8400 个预测框
┌─────────────────────┐
│ 1. 置信度过滤        │  conf < 0.25 的直接丢弃
│    8400 → ~100 个框  │  （大部分位置没有目标）
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 2. 按类别分组        │  人的框放一组，球的框放一组
│    每组独立做 NMS     │  （避免不同类别互相抑制）
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ 3. NMS 去重          │  IoU 阈值 = 0.45
│    ~100 → ~5 个框    │  每个目标只保留最佳框
└──────────┬──────────┘
           │
           ▼
       最终检测结果
```

**代码调用**（ultralytics 内部自动完成）：

```python
from ultralytics import YOLO

model = YOLO('yolov8n-pose.pt')
results = model(frame, conf=0.25, iou=0.45)
#                      ↑ 置信度阈值  ↑ NMS IoU 阈值

# 调整 NMS 参数
results = model(frame, conf=0.1, iou=0.7)   # 更宽松
results = model(frame, conf=0.5, iou=0.3)   # 更严格
```

## 在网球项目中的应用

### 场景分析

网球场景的特点：
- 通常只有 **1~2 个人**（选手 + 可能的对手/裁判）
- 人与人**不太会重叠**（场地大）
- 所以标准 NMS + 默认阈值就足够了

### 我们项目中的使用

```python
# 学习/正手教练.py 中的关键代码
model = YOLO("yolov8n-pose.pt")
results = model(frame)

# YOLO 内部已经做了 NMS，results 中每个人只有一个框
for r in results:
    if r.keypoints is not None:
        kp = r.keypoints.xy[0]   # 取第一个人的关键点
        # ... 后续处理
```

### NMS 参数对检测的影响

```
conf=0.25, iou=0.45 (默认)  →  正常检测，每人一个框
conf=0.10, iou=0.70         →  更多框被保留（可能有重复）
conf=0.50, iou=0.30         →  只保留高置信度、严格去重
```

## NMS 的优缺点总结

### 优点
- **简单直观**：逻辑清晰，易于实现和理解
- **计算高效**：O(N²) 复杂度，N 通常很小（过滤后）
- **效果好**：在大多数场景下都能有效去重

### 缺点
- **硬阈值**：IoU 阈值是一刀切的，不够灵活
- **密集目标问题**：两个真实目标重叠时，一个可能被错误删除
- **贪心算法**：按置信度排序是贪心的，不一定全局最优
- **顺序依赖**：排序结果影响最终保留的框

### 什么时候 NMS 会出问题

```
✓ 适合:  网球场景（人少、不重叠）
✓ 适合:  自动驾驶（车辆间有间距）
✗ 困难:  拥挤人群（人与人高度重叠）
✗ 困难:  货架商品（密集排列）
```
