# 损失函数与 IoU

## 概述

损失函数（Loss Function）是机器学习的**核心驱动力**。它告诉模型"你的预测离正确答案有多远"，模型通过反向传播最小化这个距离来学习。

> **类比**：想象你蒙着眼投篮。每次投完，有人告诉你"偏左了 30 厘米、偏高了 10 厘米"（这就是损失函数），你就根据这个信息调整下一次投篮的方向和力度（这就是梯度下降）。如果没人告诉你偏了多少，你就无法改进。

在目标检测中，模型需要同时完成三件事：
1. **画框** → 需要框回归损失
2. **分类** → 需要分类损失
3. **标关键点** → 需要关键点损失

---

## 什么是损失函数

### 最简单的损失：均方误差 (MSE)

```
Loss = (预测值 - 真实值)²
```

**手算示例**：预测温度 = 28°C，真实温度 = 25°C

```
Loss = (28 - 25)² = 9
```

如果预测 = 25.5°C：
```
Loss = (25.5 - 25)² = 0.25    ← 更小，说明预测更准
```

### 为什么要"平方"

| 做法 | 公式 | 预测=28 | 预测=22 | 特点 |
|------|------|---------|---------|------|
| 直接差 | y - ŷ | +3 | -3 | 正负抵消，平均可能为 0 |
| 绝对值 | \|y - ŷ\| | 3 | 3 | 在 0 处不可导 |
| **平方** | (y - ŷ)² | 9 | 9 | ✓ 始终为正 ✓ 处处可导 ✓ 大误差惩罚更重 |

## IoU (Intersection over Union)

### 为什么需要 IoU

在目标检测中，我们需要衡量**预测框和真实框有多重合**。直接比较 (x, y, w, h) 四个参数不直观，IoU 是一个统一的度量：

```
IoU = 交集面积 / 并集面积
```

### 手算 IoU 示例

```
预测框 A (50,50)→(150,150):        真实框 B (80,80)→(180,180):

     50       80      150     180
  50  ┌────────┬────────┐
      │  A     │        │
  80  │        ┌────────┼────────┐
      │        │ 交集区  │  B     │
 150  └────────┼────────┘        │
               │                 │
 180           └─────────────────┘
```

**第一步：计算交集面积**
```
交集左上角 x = max(50, 80) = 80
交集左上角 y = max(50, 80) = 80
交集右下角 x = min(150, 180) = 150
交集右下角 y = min(150, 180) = 150

交集宽 = 150 - 80 = 70
交集高 = 150 - 80 = 70
交集面积 = 70 × 70 = 4,900
```

**第二步：计算并集面积**
```
A 面积 = (150-50) × (150-50) = 100 × 100 = 10,000
B 面积 = (180-80) × (180-80) = 100 × 100 = 10,000
并集面积 = A + B - 交集 = 10,000 + 10,000 - 4,900 = 15,100
```

**第三步：计算 IoU**
```
IoU = 4,900 / 15,100 = 0.3245
```

### IoU 直觉

| IoU 值 | 重合程度 | 检测质量 |
|--------|----------|----------|
| 0.0 | 完全不重叠 | 完全错误 |
| 0.3 | 小部分重叠 | 较差 |
| **0.5** | **约一半重叠** | **通常认为"正确检测"的阈值** |
| 0.75 | 大部分重叠 | 良好 |
| 1.0 | 完全重合 | 完美 |

## 从 IoU 到 IoU 损失——逐步改进

### 第 1 代：IoU Loss

```
Loss_IoU = 1 - IoU
```

**问题**：当两个框完全不重叠时，IoU = 0，Loss = 1，**梯度为 0**！模型不知道该往哪个方向移动。

```
情况 1:  框差得远              情况 2:  框差得更远
┌──┐            ┌──┐         ┌──┐                        ┌──┐
│A │            │B │         │A │                        │B │
└──┘            └──┘         └──┘                        └──┘

IoU = 0, Loss = 1             IoU = 0, Loss = 1
                              ↑ 明明差更远，但 Loss 一样！无法区分！
```

### 第 2 代：GIoU Loss (Generalized IoU)

引入**最小包围框 C**（刚好能把 A 和 B 都装进去的最小矩形）：

```
GIoU = IoU - (C的面积 - 并集面积) / C的面积
Loss_GIoU = 1 - GIoU
```

```
┌─────────────────────────┐ ← C：最小包围框
│  ┌──┐    空白区    ┌──┐ │
│  │A │              │B │ │
│  └──┘              └──┘ │
└─────────────────────────┘

C 面积 = 很大
空白区 = C - A ∪ B = 很大
GIoU = 0 - 大/大 = 负数
→ 框越远，GIoU 越负，Loss 越大 → 有梯度了！
```

**手算 GIoU**：
```
A = (50,50)→(100,100), B = (200,200)→(250,250)  (完全不重叠)

IoU = 0
C = (50,50)→(250,250), C面积 = 200×200 = 40,000
A∪B = 50×50 + 50×50 = 5,000
GIoU = 0 - (40,000 - 5,000) / 40,000 = -0.875
Loss = 1 - (-0.875) = 1.875
```

### 第 3 代：DIoU Loss (Distance IoU)

GIoU 收敛慢，因为它需要先让框重叠。DIoU 直接**惩罚中心点距离**：

```
DIoU = IoU - d² / c²

d = 两框中心点的欧氏距离
c = 最小包围框的对角线长度
```

```
┌──────────────────────────┐
│          c（对角线）       │
│  ┌──┐──d──┌──┐           │
│  │A │     │B │           │
│  └──┘     └──┘           │
└──────────────────────────┘

d²/c² 越大 → 中心越远 → 惩罚越重
```

**优势**：直接优化中心点距离，收敛快得多。

### 第 4 代：CIoU Loss (Complete IoU) ← YOLOv8 使用

在 DIoU 基础上增加**长宽比一致性**惩罚：

```
CIoU = IoU - d²/c² - αv

v = (4/π²) × (arctan(w_gt/h_gt) - arctan(w_pred/h_pred))²
α = v / ((1-IoU) + v)
```

**含义**：即使中心点重合、IoU 很高，如果预测框的长宽比和真实框不一致（比如真实框是竖长方形，预测框是横长方形），CIoU 也会给惩罚。

### IoU 家族对比

| 方法 | 惩罚内容 | 不重叠时有梯度 | 收敛速度 |
|------|----------|---------------|----------|
| IoU | 仅重叠面积 | ✗ | 慢 |
| GIoU | + 空白区域 | ✓ | 较慢 |
| DIoU | + 中心距离 | ✓ | 快 |
| **CIoU** | **+ 中心距离 + 长宽比** | **✓** | **最快** |

## 目标检测的三大损失函数

目标检测需要同时完成三个任务，所以**总损失是三个子损失的加权和**：

```
Total Loss = λ₁ × Box Loss + λ₂ × Cls Loss + λ₃ × Keypoint Loss
```

> 想象你同时学打网球的三个方面：发力（框）、落点判断（分类）、握拍姿势（关键点）。教练给三方面分别打分，加权求总分。

### 1. 边界框损失 (Box Loss) —— CIoU

**任务**：让预测框尽量和真实框重合。

```python
import numpy as np

def calculate_iou(box1, box2):
    """box 格式: [x1, y1, x2, y2]"""
    # 交集
    x1 = max(box1[0], box2[0])
    y1 = max(box1[1], box2[1])
    x2 = min(box1[2], box2[2])
    y2 = min(box1[3], box2[3])
    intersection = max(0, x2 - x1) * max(0, y2 - y1)
    
    # 并集
    area1 = (box1[2] - box1[0]) * (box1[3] - box1[1])
    area2 = (box2[2] - box2[0]) * (box2[3] - box2[1])
    union = area1 + area2 - intersection
    
    return intersection / union if union > 0 else 0

def ciou_loss(box_pred, box_true):
    iou = calculate_iou(box_pred, box_true)
    
    # 中心点
    cx_p = (box_pred[0] + box_pred[2]) / 2
    cy_p = (box_pred[1] + box_pred[3]) / 2
    cx_t = (box_true[0] + box_true[2]) / 2
    cy_t = (box_true[1] + box_true[3]) / 2
    d2 = (cx_p - cx_t)**2 + (cy_p - cy_t)**2   # 中心距离²
    
    # 最小包围框对角线
    enclose_x1 = min(box_pred[0], box_true[0])
    enclose_y1 = min(box_pred[1], box_true[1])
    enclose_x2 = max(box_pred[2], box_true[2])
    enclose_y2 = max(box_pred[3], box_true[3])
    c2 = (enclose_x2 - enclose_x1)**2 + (enclose_y2 - enclose_y1)**2  # 对角线²
    
    # 长宽比惩罚
    w_p = box_pred[2] - box_pred[0]
    h_p = box_pred[3] - box_pred[1]
    w_t = box_true[2] - box_true[0]
    h_t = box_true[3] - box_true[1]
    v = (4 / np.pi**2) * (np.arctan(w_t/h_t) - np.arctan(w_p/h_p))**2
    alpha = v / (1 - iou + v + 1e-8)
    
    ciou = iou - d2/c2 - alpha * v
    return 1 - ciou
```

### 2. 分类损失 (Classification Loss) —— 二元交叉熵 (BCE)

**任务**：让模型正确判断"这个框里是人还是背景"。

**二元交叉熵公式**：
```
BCE = -[ y × log(ŷ) + (1-y) × log(1-ŷ) ]

y = 真实标签 (0 或 1)
ŷ = 模型预测的概率 (0~1)
```

**手算示例**：

```
场景 1: 真的是人 (y=1), 模型预测 ŷ=0.9 (很确信)
BCE = -[1×log(0.9) + 0×log(0.1)]
    = -log(0.9) = 0.105                 ← 损失很小 ✓

场景 2: 真的是人 (y=1), 模型预测 ŷ=0.1 (不确信)
BCE = -[1×log(0.1) + 0×log(0.9)]
    = -log(0.1) = 2.303                 ← 损失很大 ✗ 需要调整！

场景 3: 不是人 (y=0), 模型预测 ŷ=0.9 (误报)
BCE = -[0×log(0.9) + 1×log(0.1)]
    = -log(0.1) = 2.303                 ← 损失很大 ✗ 需要调整！
```

**直觉**：
- 预测对了且很确信 → 损失接近 0
- 预测错了 → 损失急剧增大（log 的惩罚效果）
- 越"自信地犯错"，惩罚越重

### 3. 关键点损失 (Keypoint Loss) —— OKS

**任务**：让预测的关键点坐标尽量接近真实位置。

```python
def keypoint_loss(pred_kp, true_kp, visibility):
    """
    pred_kp: 预测关键点 (17, 2)
    true_kp: 真实关键点 (17, 2)
    visibility: 可见性 (17,)  1=可见, 0=被遮挡
    """
    # 逐关键点计算距离
    diff = pred_kp - true_kp             # (17, 2)
    dist_sq = (diff ** 2).sum(axis=1)    # (17,) 每个关键点的距离²
    
    # 只对可见关键点计算损失
    loss = (dist_sq * visibility).sum() / (visibility.sum() + 1e-8)
    return loss
```

**手算示例**：
```
真实左肩 = (200, 120)，预测左肩 = (205, 118)，可见
距离² = (205-200)² + (118-120)² = 25 + 4 = 29

真实左腕 = (350, 280)，预测左腕 = (340, 285)，可见
距离² = (340-350)² + (285-280)² = 100 + 25 = 125

真实右腕 = (0, 0)，被遮挡，visibility=0 → 不参与计算

Loss = (29×1 + 125×1 + 0×0) / (1+1+0) = 154/2 = 77
```

## 总损失的平衡

### 为什么需要权重 λ

三个损失的数值范围可能差很多：
```
Box Loss ≈ 0.5 ~ 2.0
Cls Loss ≈ 0.1 ~ 5.0
Kpt Loss ≈ 10 ~ 500
```

如果直接相加，关键点损失会**主导**训练，模型只优化关键点，忽略框和分类。

**解决方案**：用权重平衡

```
Total = 7.5 × Box + 0.5 × Cls + 1.0 × Kpt    (YOLOv8 默认权重)
```

### 损失曲线怎么看

```
Loss ↑
  4 │ ╲
    │   ╲
  3 │    ╲  训练 Loss
    │     ╲──────────────
  2 │      ╲               验证 Loss
    │       ──────────── ╱──────
  1 │                  ╱
    │                 这里开始过拟合！
  0 ├──┬──┬──┬──┬──┬──┬──→ Epoch
    0  10 20 30 40 50 60
```

- **训练和验证 Loss 都下降** → 模型在学习 ✓
- **训练 Loss 下降，验证 Loss 上升** → 过拟合 ✗（应该早停）
- **训练 Loss 不下降** → 学习率太小，或模型容量不够

## 在网球项目中的应用

### LSTM 正手检测的损失

在我们的网球项目中，LSTM 模型使用**二元交叉熵 (BCE)** 判断滑动窗口是否为正手动作：

```python
# 参考代码: 学习/正手检测器_LSTM.py
criterion = nn.BCELoss()

# 前向传播
pred = model(x_batch)        # 预测概率, 如 0.87
loss = criterion(pred, y)    # y=1(正手) 或 y=0(非正手)

# 反向传播
loss.backward()
optimizer.step()
```

### YOLOv8 姿态估计的损失

YOLO 训练时同时优化三个损失：

```
每帧图像 → YOLO 前向传播 → 预测 (框 + 类别 + 关键点)
                              │
                              ▼
                         与标注对比
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
         CIoU Loss       BCE Loss        OKS Loss
         (框回归)         (分类)          (关键点)
              │               │               │
              └───────┬───────┘               │
                      ▼                       │
              Total = λ₁×Box + λ₂×Cls + λ₃×Kpt
                      │
                      ▼
                 反向传播更新权重
```

参考脚本：`学习/A3_看IoU损失.py`

```python
import numpy as np

# 手动计算 IoU
box_pred = [50, 50, 150, 150]
box_true = [80, 80, 180, 180]

iou = calculate_iou(box_pred, box_true)
print(f"IoU = {iou:.4f}")       # 0.3245
print(f"Loss = {1 - iou:.4f}")  # 0.6755

# 尝试不同偏移，观察 IoU 变化
for dx in range(0, 101, 10):
    shifted = [50+dx, 50+dx, 150+dx, 150+dx]
    iou = calculate_iou(shifted, box_true)
    print(f"偏移 {dx:3d} px → IoU = {iou:.4f}, Loss = {1-iou:.4f}")
```
