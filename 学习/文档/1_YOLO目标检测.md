# YOLO 目标检测详解

## 概述

YOLO (You Only Look Once) 是一种**单阶段目标检测算法**。在 YOLO 出现之前，主流的目标检测方法是"两阶段"的：
1. 先用一个网络提出候选区域（"这里可能有东西"）
2. 再用另一个网络对每个候选区域分类（"这是什么东西"）

这种做法准确但**非常慢**。YOLO 的革命性在于——**只看一次**：用一个神经网络，一次前向传播，同时预测所有目标的位置和类别。

> **类比**：传统方法就像你在教室里先扫一遍找到所有人，再一个一个认脸；YOLO 就像你扫一眼就同时知道每个人是谁、坐在哪。

---

## 核心思想：网格 + 回归

### 第一步：把图像切成网格

假设输入图像大小是 640×640 像素，YOLO 会把它划分为 S×S 的网格（比如 20×20 = 400 个格子）。

```
┌───┬───┬───┬───┬───┐
│   │   │   │   │   │
├───┼───┼───┼───┼───┤
│   │   │ ★ │   │   │  ← 人物中心落在这个格子
├───┼───┼───┼───┼───┤
│   │   │   │   │   │
├───┼───┼───┼───┼───┤
│   │   │   │   │   │
└───┴───┴───┴───┴───┘
```

**规则**：哪个格子包含目标的**中心点**，哪个格子就负责预测这个目标。

### 第二步：每个格子预测什么

每个格子输出一个向量，包含：

| 参数 | 含义 | 范围 |
|------|------|------|
| x, y | 边界框中心相对于格子左上角的偏移 | [0, 1] |
| w, h | 边界框宽高相对于整张图的比例 | [0, 1] |
| conf | 置信度 = P(有目标) × IoU | [0, 1] |
| c₁, c₂, ... | 各类别概率 | [0, 1] |

**举例**：假设图像 640×640，格子大小 32×32，某格子左上角坐标 (128, 96)。
如果预测 x=0.6, y=0.3, w=0.15, h=0.4，那么：
- 框中心 = (128 + 0.6×32, 96 + 0.3×32) = (147.2, 105.6) 像素
- 框宽 = 0.15 × 640 = 96 像素
- 框高 = 0.4 × 640 = 256 像素

### 第三步：选出最终检测结果

每张图可能产生数千个预测框，通过以下步骤筛选：
1. **置信度过滤**：丢弃 conf < 0.25 的框
2. **NMS（非极大值抑制）**：去除重复框（详见 NMS 文档）

## YOLO 的演进历史

| 版本 | 年份 | 关键改进 |
|------|------|----------|
| YOLOv1 | 2016 | 开创性地提出单阶段检测 |
| YOLOv2 | 2017 | 引入 anchor boxes、多尺度训练 |
| YOLOv3 | 2018 | 多尺度预测（3 个尺度）、Darknet-53 |
| YOLOv4 | 2020 | CSPDarknet、Mish 激活、数据增强 |
| YOLOv5 | 2020 | PyTorch 实现、自动 anchor |
| YOLOv8 | 2023 | **Anchor-Free**、解耦头、Pose 扩展 ← 我们使用的版本 |

## YOLOv8 架构详解

### 整体流程

```
输入图像 (640×640×3)
    │
    ▼
┌──────────────┐
│   Backbone   │   CSPDarknet：提取图像特征
│  (主干网络)   │   通过卷积层层层提取，从边缘→纹理→形状→语义
└──────┬───────┘
       │  输出 3 个不同尺度的特征图
       ▼
┌──────────────┐
│     Neck     │   PAFPN：特征金字塔 + 路径聚合
│  (颈部网络)   │   融合不同尺度的特征
└──────┬───────┘
       │  3 个融合后的特征图
       ▼
┌──────────────┐
│     Head     │   解耦头：分别预测框和类别
│  (检测头)     │   + 关键点回归（Pose 版本）
└──────────────┘
       │
       ▼
   检测结果: [x, y, w, h, conf, class, 17×(kp_x, kp_y, kp_conf)]
```

### 1. Backbone（主干网络）—— CSPDarknet

主干网络的作用是**从原始像素中提取有意义的特征**。

```
输入 640×640×3
    │
    ├─→ Conv 3×3, stride=2  →  320×320×32    (边缘、颜色)
    ├─→ C2f Block           →  160×160×64    (纹理、角点)
    ├─→ C2f Block           →  80×80×128     (局部形状)   ← P3 小尺度特征
    ├─→ C2f Block           →  40×40×256     (物体部件)   ← P4 中尺度特征
    └─→ C2f Block + SPPF    →  20×20×512     (整体语义)   ← P5 大尺度特征
```

- **浅层** (P3, 80×80)：分辨率高，保留位置细节，适合**检测小目标**
- **深层** (P5, 20×20)：分辨率低，但语义信息丰富，适合**检测大目标**

> **C2f Block**：跨阶段部分（Cross Stage Partial）+ 瓶颈层，在减少计算量的同时保持梯度流通畅。

### 2. Neck（颈部网络）—— PAFPN

问题：浅层有好的位置信息但缺乏语义，深层有好的语义但位置粗糙。

**解决方案**：特征金字塔网络（FPN）+ 路径聚合网络（PAN）

```
        P5 (20×20)                    P5' (20×20)
           │                              ▲
     ┌─ 上采样 ─┐                    ┌─ 下采样 ─┐
     ▼          │                    │          ▼
   拼接 P4 → 融合 → P4' (40×40)  → 融合 → P4'' (40×40)
     │                               │
     ┌─ 上采样 ─┐                    ┌─ 下采样 ─┐
     ▼          │                    │          ▼
   拼接 P3 → 融合 → P3' (80×80)  → 融合 → P3'' (80×80)

   FPN（自顶向下，传递语义）    PAN（自底向上，传递位置）
```

每个尺度的特征图都同时拥有**精确的位置信息**和**丰富的语义信息**。

### 3. Head（检测头）—— 解耦头 + Anchor-Free

YOLOv8 的检测头将**分类**和**回归**分成两个独立分支：

```
特征图 (如 80×80×128)
    │
    ├──→ 回归分支：Conv → Conv → 预测 (x, y, w, h) + DFL
    │
    └──→ 分类分支：Conv → Conv → 预测 类别概率
    
    (Pose 版本额外增加)
    └──→ 关键点分支：Conv → Conv → 预测 17×(kp_x, kp_y, kp_conf)
```

**Anchor-Free 是什么意思？**

| 特性 | Anchor-Based (旧) | Anchor-Free (YOLOv8) |
|------|-------------------|----------------------|
| 预定义框 | 需要手动设计多种大小的锚框 | 不需要 |
| 预测方式 | 预测锚框的偏移量 | 直接预测框的位置 |
| 灵活性 | 受限于锚框设计 | 更自由 |
| 超参数 | 需要调节锚框尺寸 | 更少 |

## 姿态估计扩展 (YOLOv8-Pose)

在普通目标检测的基础上，YOLOv8-Pose **额外回归 17 个人体关键点**。

### COCO 17 关键点定义

```
          0:鼻子
       1:左眼  2:右眼
     3:左耳      4:右耳
         ╲      ╱
      5:左肩──6:右肩
         │      │
      7:左肘  8:右肘
         │      │
      9:左腕  10:右腕
         │      │
     11:左髋──12:右髋
         │      │
     13:左膝  14:右膝
         │      │
     15:左踝  16:右踝
```

### 关键点预测输出

对于每个检测到的人，模型输出：
- **17 × 3 = 51 个数值**：每个关键点的 (x, y, confidence)
- x, y：关键点在图像中的像素坐标
- confidence：该关键点是否可见（被遮挡时接近 0）

### 实际输出数据示例

```python
# 假设检测到一个网球运动员
result = model(frame)[0]

# 边界框
box = result.boxes[0]
print(box.xyxy)   # tensor([[120, 50, 450, 600]])  左上角(120,50) 右下角(450,600)
print(box.conf)   # tensor([0.95])                  95% 确信是人

# 关键点
kp = result.keypoints[0]
print(kp.xy.shape)    # torch.Size([1, 17, 2])     17个关键点的(x,y)
print(kp.conf.shape)  # torch.Size([1, 17])         17个关键点的置信度

# 具体数值
print(kp.xy[0, 5])    # tensor([200, 120])  左肩坐标
print(kp.xy[0, 9])    # tensor([350, 280])  左腕坐标（击球手）
print(kp.conf[0, 9])  # tensor([0.88])       左腕置信度 88%
```

## 可视化代码

参考脚本：`学习/A1_看YOLO输出.py`

```python
from ultralytics import YOLO
import cv2

model = YOLO('yolov8n-pose.pt')
results = model('path/to/tennis_frame.jpg')

for r in results:
    # 绘制边界框
    boxes = r.boxes
    for box in boxes:
        x1, y1, x2, y2 = box.xyxy[0]
        conf = box.conf[0]
        cv2.rectangle(img, (int(x1), int(y1)), (int(x2), int(y2)), (0, 255, 0), 2)
        cv2.putText(img, f'{conf:.2f}', (int(x1), int(y1)-5),
                    cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 1)
    
    # 绘制关键点
    keypoints = r.keypoints
    for kp in keypoints.xy:
        for i, (x, y) in enumerate(kp):
            if x > 0 and y > 0:  # 只绘制有效关键点
                cv2.circle(img, (int(x), int(y)), 3, (0, 0, 255), -1)
    
    # 绘制骨架连线
    skeleton = [(5,7),(7,9),(6,8),(8,10),   # 手臂
                (5,6),(11,12),               # 躯干
                (5,11),(6,12),               # 躯干两侧
                (11,13),(13,15),             # 左腿
                (12,14),(14,16)]             # 右腿
    for (a, b) in skeleton:
        x1, y1 = int(kp[a][0]), int(kp[a][1])
        x2, y2 = int(kp[b][0]), int(kp[b][1])
        if x1 > 0 and y1 > 0 and x2 > 0 and y2 > 0:
            cv2.line(img, (x1, y1), (x2, y2), (255, 255, 0), 2)
```

## 优势与局限

### 优势
- **速度极快**：YOLOv8n 在 GPU 上可达 **200+ FPS**，适合实时视频
- **端到端**：输入图像 → 直接输出框+类别+关键点，无需复杂流水线
- **部署方便**：支持导出为 ONNX、TensorRT、CoreML 等格式

### 局限
- **小目标检测**：对远处的小人物（如观众席）敏感度较低
- **遮挡问题**：严重遮挡时关键点可能不准
- **定位精度**：边界框精度略低于两阶段方法（但差距越来越小）
- **密集场景**：多人紧密站在一起时可能漏检或混淆关键点

## 在网球分析中的完整应用流程

```
视频文件 (30fps)
    │
    ▼ 逐帧读取
┌─────────────────┐
│  YOLOv8-Pose    │  ← 每帧推理 ~5ms
│  检测 + 姿态     │
└────────┬────────┘
         │  输出: 17 个关键点 (x,y)
         ▼
┌─────────────────┐
│  姿态归一化      │  ← pose_norm.py
│  平移: 减髋部中心 │
│  缩放: 除躯干长度 │
└────────┬────────┘
         │  归一化后: 34 维特征向量
         ▼
┌─────────────────┐
│  标准化          │  ← 减均值 / 除标准差
│  (用训练集统计量) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  LSTM 滑动窗口   │  ← 30帧窗口 → 正手概率
│  输入: (30, 34)  │
│  输出: P(正手)    │
└────────┬────────┘
         │  概率曲线
         ▼
┌─────────────────┐
│  峰值检测 + DTW  │  ← 定位正手段落 → 与模板比较
└─────────────────┘
```

1. **YOLO 提取关键点**：从每帧画面中检测人物并提取 17 个身体关键点
2. **姿态归一化**：将关键点坐标归一化，消除位置和体型差异
3. **LSTM 时序分类**：用 30 帧的关键点序列判断是否在执行正手动作
4. **动作分析**：检测到正手后，通过 DTW 与标准模板比较，给出诊断
