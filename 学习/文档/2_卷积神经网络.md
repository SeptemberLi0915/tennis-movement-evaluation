# 卷积神经网络 (CNN) 原理

## 概述

卷积神经网络 (Convolutional Neural Network, CNN) 是一种专门处理网格状数据（如图像）的深度学习架构。

> **类比**：想象你拿着一个小放大镜在一张大照片上逐格扫描。每扫一格，你只看放大镜框内的一小块区域，记录下这块区域的某种特征（比如"这里有一条竖线"）。扫完整张照片后，你就得到了一张"特征地图"。这就是卷积操作的本质。

在传统的全连接神经网络中，如果输入一张 640×640×3 的彩色图像，就有 640×640×3 = **1,228,800** 个输入节点，如果隐藏层有 1000 个神经元，仅第一层就需要 **12 亿** 个参数——这是不现实的。CNN 通过**局部连接**和**权重共享**两个巧妙设计，将参数量降低到几千个。

## 卷积操作

### 基本原理

卷积操作使用一个小矩阵（称为"卷积核" kernel，或"滤波器" filter），在输入图像上**逐位置滑动**，每个位置计算**逐元素相乘再求和**。

### 手算示例

假设输入是一个 5×5 的灰度图像，卷积核是 3×3：

```
输入图像 (5×5)              卷积核 (3×3)
┌───┬───┬───┬───┬───┐      ┌────┬────┬────┐
│ 1 │ 2 │ 3 │ 0 │ 1 │      │ 1  │ 0  │ -1 │
├───┼───┼───┼───┼───┤      ├────┼────┼────┤
│ 0 │ 1 │ 2 │ 3 │ 1 │      │ 1  │ 0  │ -1 │
├───┼───┼───┼───┼───┤      ├────┼────┼────┤
│ 1 │ 0 │ 1 │ 0 │ 2 │      │ 1  │ 0  │ -1 │
├───┼───┼───┼───┼───┤      └────┴────┴────┘
│ 2 │ 1 │ 0 │ 1 │ 1 │
├───┼───┼───┼───┼───┤
│ 1 │ 0 │ 2 │ 1 │ 0 │
└───┴───┴───┴───┴───┘
```

**第一步**：把卷积核放在左上角 (0,0) 位置，对齐 3×3 区域：

```
选中区域:           逐元素相乘:
1  2  3             1×1  2×0  3×(-1)    =  1  0  -3
0  1  2             0×1  1×0  2×(-1)    =  0  0  -2
1  0  1             1×1  0×0  1×(-1)    =  1  0  -1

求和: 1 + 0 + (-3) + 0 + 0 + (-2) + 1 + 0 + (-1) = -4
```

**输出 (0,0) = -4**

**第二步**：卷积核右移一格到 (0,1)，选中新的 3×3 区域，重复计算...

```
选中区域:           逐元素相乘:
2  3  0             2×1  3×0  0×(-1)    =  2  0   0
1  2  3             1×1  2×0  3×(-1)    =  1  0  -3
0  1  0             0×1  1×0  0×(-1)    =  0  0   0

求和: 2 + 0 + 0 + 1 + 0 + (-3) + 0 + 0 + 0 = 0
```

**输出 (0,1) = 0**

重复此过程，最终得到 3×3 的输出特征图（因为 5-3+1 = 3）：

```
输出特征图 (3×3)
┌────┬────┬────┐
│ -4 │  0 │  4 │
├────┼────┼────┤
│ -3 │  1 │  2 │
├────┼────┼────┤
│  0 │  2 │ -1 │
└────┴────┴────┘
```

### 输出尺寸公式

```
输出大小 = (输入大小 - 卷积核大小 + 2×填充) / 步长 + 1
```

**举例**：
- 输入 5×5，核 3×3，padding=0，stride=1 → 输出 (5-3+0)/1+1 = **3×3**
- 输入 32×32，核 3×3，padding=1，stride=1 → 输出 (32-3+2)/1+1 = **32×32**（尺寸不变！）
- 输入 32×32，核 3×3，padding=0，stride=2 → 输出 (32-3+0)/2+1 = **15×15**（缩小一半）

### 关键参数详解

| 参数 | 含义 | 直觉理解 |
|------|------|----------|
| **核大小 (Kernel Size)** | 3×3, 5×5, 7×7 等 | 放大镜的大小，越大看到的范围越广 |
| **步长 (Stride)** | 每次滑动几个像素 | 步长 2 = 隔一格扫一次 → 输出缩小一半 |
| **填充 (Padding)** | 在边缘补 0 | padding=1 + kernel=3 → 输出和输入同大小 |
| **通道数 (Channels)** | 输入/输出的深度 | 彩色图像有 RGB 3 个通道 |

## 经典卷积核效果

### 1. 边缘检测核 (Sobel)——横向边缘 vs 纵向边缘

#### 先分清"边缘的方向"

- **纵向边缘（竖直边缘 vertical edge）**：边缘线本身是**竖着**的。例：球拍杆、门框左右两条边、人体躯干的左右轮廓。
  → 此处图像亮度沿**水平方向（左右）**发生突变（左暗右亮 或 左亮右暗）。
- **横向边缘（水平边缘 horizontal edge）**：边缘线本身是**横着**的。例：球场底线、网带上沿、地平线、肩膀的上边缘。
  → 此处图像亮度沿**垂直方向（上下）**发生突变（上暗下亮 或 上亮下暗）。

> 一句话记牢：**边缘的方向 ⊥ 亮度突变的方向**。竖直的边，是"左右在变"；水平的边，是"上下在变"。

#### 核心原理：卷积核在哪个方向"做差分"，就测那个方向的亮度变化

Sobel 核本质是**方向导数**（差分）：一侧取正、另一侧取负，算的是"这一侧减另一侧"的亮度差。

**Gx（检测纵向/竖直边缘）**——左列负、右列正，做的是"右 − 左"的**水平**差分：

```
┌────┬────┬────┐
│ -1 │  0 │  1 │
├────┼────┼────┤
│ -2 │  0 │  2 │   ← 中间行权重更大，抗噪
├────┼────┼────┤
│ -1 │  0 │  1 │
└────┴────┴────┘
```
- 遇到"左暗右亮"的**竖直边** → 右列正数乘到大值、左列负数乘到小值 → 输出**大正数**。
- 左右一样亮（平坦区）→ 正负抵消 → 输出 **≈ 0**。

**Gy（检测横向/水平边缘）**——上行负、下行正，做的是"下 − 上"的**垂直**差分：

```
┌────┬────┬────┐
│ -1 │ -2 │ -1 │
├────┼────┼────┤
│  0 │  0 │  0 │
├────┼────┼────┤
│  1 │  2 │  1 │
└────┴────┴────┘
```
- 遇到"上暗下亮"的**水平边** → 输出大值；上下一样亮 → 输出 ≈ 0。

#### ⚠️ 最容易搞反的命名点

`Gx / Sobel_x` 名字里的 **x 指"求 x（水平）方向的梯度"**，而它**检测到的却是竖直边缘**。
原因就是上面那句：**做差分的方向 = 梯度方向；检测到的边缘方向 ⊥ 梯度方向**。
- Gx 沿水平做差 → 抓**竖直**边
- Gy 沿垂直做差 → 抓**水平**边

#### 手算验证：一张"左黑右白"的竖直边图像

```
输入（左两列=0 暗，右三列=9 亮）        用 Gx 在边界处取 3×3：
0 0 9 9 9                                0 0 9        (-1·0 + 0·0 + 1·9)
0 0 9 9 9      ← 中间有一条竖直边        0 0 9   →    (-2·0 + 0·0 + 2·9)   = 9+18+9 = 36  ✅ 强响应
0 0 9 9 9                                0 0 9        (-1·0 + 0·0 + 1·9)

平坦区（全 5）：                          用 Gy 在同一竖直边处：
5 5 5                                     0 0 9        (-1·0 -2·0 -1·9)
5 5 5   →  Gx = 0  ❌ 无边               0 0 9   →    ( 0    0    0  )    = -9 +9 = 0  ❌ 不响应
5 5 5                                     0 0 9        ( 1·0  2·0  1·9)
```
- Gx 在竖直边处输出 **36**（强），在平坦区输出 **0** → 它就是"竖直边探测器"。
- Gy 对这条**竖直**边输出 **0** → 它对竖直边"视而不见"，专抓水平边。

#### 两个方向合起来：任意方向的边

单独一个 Gx 或 Gy 只抓一种朝向的边。把两者结合，就能得到**任意方向**边缘的强度和角度：

```
梯度幅值（边有多强）: G  = sqrt(Gx² + Gy²)
梯度方向（边朝哪转）: θ = atan2(Gy, Gx)
```

这正是你 `A2_看卷积.py`（若存在）里 `edges = sqrt(edge_x² + edge_y²)` 在做的事——把水平、竖直两张边缘图合成一张完整轮廓图。

**应用在网球图像上**：
- **Gx（竖直边）**：球拍杆、身体左右轮廓、球网竖柱
- **Gy（水平边）**：球场底线/发球线、网带上下沿、肩膀上缘
- **合并 G**：完整勾出球拍、人体、球场线的轮廓

### 2. 锐化核

```
┌────┬────┬────┐
│  0 │ -1 │  0 │
├────┼────┼────┤
│ -1 │  5 │ -1 │
├────┼────┼────┤
│  0 │ -1 │  0 │
└────┴────┴────┘
```

**直觉**：中心权重 5，周围是 -1。如果像素和周围一样，5×v - 4×v = v（保持不变）；如果像素比周围亮，5×v - 4×小值 > v（变得更亮）。效果：**增强对比度和细节**。

### 3. 高斯模糊核

```
┌──────┬──────┬──────┐
│ 1/16 │ 2/16 │ 1/16 │
├──────┼──────┼──────┤
│ 2/16 │ 4/16 │ 2/16 │
├──────┼──────┼──────┤
│ 1/16 │ 2/16 │ 1/16 │
└──────┴──────┴──────┘
```

**直觉**：对每个像素取周围的加权平均，中心权重最大。效果：**平滑噪声**。

## CNN 的核心设计哲学

### 1. 局部感受野 (Local Receptive Field)

**问题**：全连接层中，每个神经元连接所有输入像素 → 参数爆炸。

**解决**：每个神经元只看输入的一小块区域（3×3 或 5×5），就像用放大镜只看一小部分。

```
全连接 (每个都连):              卷积 (局部连接):
输入: ● ● ● ● ● ● ● ● ●      输入: ● ● ● ● ● ● ● ● ●
       ╲╲╲╲│╱╱╱╱                        ╲│╱
输出:        ●                  输出:     ●
```

### 2. 权重共享 (Weight Sharing)

**问题**：即使局部连接，不同位置用不同权重仍然参数很多。

**解决**：整张图共用同一组权重（同一个卷积核）。

**参数量对比**：
- 全连接（输入 1000，输出 1000）：1000 × 1000 = **1,000,000 参数**
- 卷积（3×3 核，64 个输出通道）：3 × 3 × 64 = **576 参数**

### 3. 层次化特征 (Hierarchical Features)

```
第 1 层卷积 → 边缘、角点
    ↓
第 2 层卷积 → 纹理、简单形状
    ↓
第 3 层卷积 → 物体部件（手臂、球拍头）
    ↓
第 4 层卷积 → 整体目标（人、球拍、网球）
```

> **类比**：就像画画——先画线条（边缘），再画形状（纹理），然后画部件（手臂），最后画整体（人物）。

## CNN 架构组件详解

### 1. 卷积层 (Convolutional Layer)

#### 权重张量的形状——从 2D 到 4D

前面手算示例用的是灰度图 + 单个 3×3 核，权重只是一个 2D 矩阵。但真实 CNN 里，输入有多个通道、输出也有多个通道，权重就变成了一个 **4D 张量**。下面一步步推出来：

**① 灰度图 + 1 个核（你已经会了）**

输入 1 通道，核是 3×3 矩阵，滑动点积 → 输出 1 张特征图。权重 shape = `(3, 3)`，共 9 个参数。

**② 彩色图（RGB 3 通道）+ 1 个滤波器**

输入有 R、G、B 三层。一个 3×3 矩阵只能看一层，不够。所以一个滤波器 = **3 片 3×3 矩阵叠在一起**，每片负责一个通道：

```
输入 (H × W × 3):            一个滤波器 (3片 × 3×3):       输出 (H' × W' × 1):
┌──────────────┐              ┌──────────┐
│   R 通道     │  ← 对应 →   │  核片_R   │  3×3
│   G 通道     │  ← 对应 →   │  核片_G   │  3×3     →   一张特征图
│   B 通道     │  ← 对应 →   │  核片_B   │  3×3
└──────────────┘              └──────────┘
```

计算：**每片核和对应通道做 2D 卷积**，得到 3 张中间结果，然后**逐像素相加** → 输出 1 张特征图。
滤波器 shape = `(3, 3, 3)`（输入通道数 × 核高 × 核宽），共 27 个参数。

**③ 要 32 张不同的特征图？→ 用 32 个滤波器**

每个滤波器 `(3, 3, 3)` 输出 1 张图；想要 32 张 → 堆叠 32 个滤波器。

**最终权重张量 shape = `(C_out, C_in, K_h, K_w)` = `(32, 3, 3, 3)`**

```
权重张量 shape = (C_out, C_in, K_h, K_w)
                   ↑      ↑     ↑    ↑
               输出通道  输入通道  核高  核宽
```

- `C_out = 32`：有 32 个滤波器，产出 32 张特征图
- `C_in = 3`：每个滤波器有 3 片（对应 RGB 三通道）
- `K_h × K_w = 3 × 3`：每片是 3×3 矩阵
- 参数量 = 32 × 3 × 3 × 3 = **864**（加 32 个 bias → 896）

> **一句话总结**：权重张量就是"一叠滤波器"，每个滤波器又是"一叠核片"。  
> **通道数决定核片数，滤波器个数决定输出特征图数。**

#### 代码示例

```python
import torch.nn as nn

# 输入: 3通道(RGB), 输出: 32个特征图, 核大小: 3×3
conv = nn.Conv2d(in_channels=3, out_channels=32, kernel_size=3, padding=1)

# 输入: (batch, 3, 640, 640)
# 输出: (batch, 32, 640, 640)  ← padding=1 保持尺寸

# 查看权重形状：
print(conv.weight.shape)   # → torch.Size([32, 3, 3, 3])
print(conv.bias.shape)     # → torch.Size([32])
```

**32 个输出通道**意味着 32 个不同的滤波器，每个学到不同特征：
- 滤波器 1 可能学到"检测水平线"
- 滤波器 2 可能学到"检测圆形"
- 滤波器 15 可能学到"检测肤色"
- ...

### 2. 激活函数 (Activation)

如果没有激活函数，多层卷积叠起来等价于一层（因为线性的线性还是线性）。

**ReLU**：f(x) = max(0, x)

```
输入:  [-2, 3, -1, 5, 0, -4, 7]
ReLU:  [ 0, 3,  0, 5, 0,  0, 7]  ← 负数变 0，正数不变
```

**为什么用 ReLU 而不用 Sigmoid？**
- Sigmoid 在两端梯度趋近 0（梯度消失），ReLU 在正半轴梯度恒为 1
- ReLU 计算简单：一次比较操作
- ReLU 产生稀疏激活：很多神经元输出 0，减少计算量

### 3. 池化层 (Pooling)

**作用**：降低特征图的分辨率，减少计算量，同时增强平移不变性。

**最大池化示例 (2×2, stride=2)**：
```
输入 (4×4):                    输出 (2×2):
┌───┬───┬───┬───┐             ┌───┬───┐
│ 1 │ 3 │ 2 │ 4 │             │ 5 │ 4 │  ← max(1,3,0,5)=5, max(2,4,1,4)=4
├───┼───┼───┼───┤             ├───┼───┤
│ 0 │ 5 │ 1 │ 4 │             │ 8 │ 6 │  ← max(2,8,1,3)=8, max(0,2,6,4)=6
├───┼───┼───┼───┤             └───┴───┘
│ 2 │ 8 │ 0 │ 2 │
├───┼───┼───┼───┤
│ 1 │ 3 │ 6 │ 4 │
└───┴───┴───┴───┘
```

**尺寸变化**：4×4 → 2×2（长宽各减半），但**特征图深度不变**。

**平移不变性**：目标稍微移动几个像素，池化后输出不变 → 模型更鲁棒。

### 4. 批归一化 (Batch Normalization)

在每一层卷积后，对特征值做归一化：

```
x_norm = (x - mean) / sqrt(var + ε)
y = γ × x_norm + β    ← γ, β 是可学习参数
```

**作用**：
- 加速训练收敛
- 减轻对初始化的敏感
- 起到轻微的正则化作用

### 5. 全连接层 (Fully Connected)

在最后几层，把特征图展平成一维向量，连接到分类器：

```
特征图 (7×7×512) → 展平 → 25088 维向量 → FC(4096) → FC(1000) → Softmax
                                                                    ↓
                                                              1000 个类别概率
```

## 一个完整 CNN 的前向传播示例

以处理网球图像为例：

```
输入: 网球视频帧 (640×640×3)

Conv1: 3×3, 3→32, stride=2  →  320×320×32   "检测到一些边缘和色块"
BN + ReLU
Conv2: 3×3, 32→64, stride=2 →  160×160×64   "检测到纹理和角点"
BN + ReLU
Conv3: 3×3, 64→128          →  160×160×128  "检测到手臂、球拍等部件"
BN + ReLU
MaxPool: 2×2                →  80×80×128    "缩小一半，保留重要特征"
Conv4: 3×3, 128→256         →  80×80×256    "检测到完整的人体姿态"
BN + ReLU
...

最终: 一系列特征图，每个位置包含丰富的语义信息
```

```python
import cv2
import numpy as np

# 加载图像
img = cv2.imread('tennis_frame.jpg', cv2.IMREAD_GRAYSCALE)

# 定义卷积核
sobel_x = np.array([[-1, 0, 1], [-2, 0, 2], [-1, 0, 1]])   # 检测垂直边缘
sobel_y = np.array([[-1, -2, -1], [0, 0, 0], [1, 2, 1]])     # 检测水平边缘
sharpen = np.array([[0, -1, 0], [-1, 5, -1], [0, -1, 0]])     # 锐化
blur = np.ones((3, 3)) / 9                                      # 模糊

# 应用卷积（本质上就是上面手算的过程，只不过 cv2 帮我们自动完成了）
edge_x = cv2.filter2D(img, -1, sobel_x)
edge_y = cv2.filter2D(img, -1, sobel_y)
sharp = cv2.filter2D(img, -1, sharpen)

# 合并水平和垂直边缘 → 完整边缘图
edges = np.sqrt(edge_x.astype(float)**2 + edge_y.astype(float)**2).astype(np.uint8)
```

## 卷积在 YOLO 中的具体应用

### 1. Backbone 中的卷积

YOLO 的主干网络（CSPDarknet）全部由卷积构成，没有全连接层：

```
输入 → Conv(3×3) → BN → SiLU → Conv(3×3) → BN → SiLU → ... → 特征图
```

- **浅层卷积**：学到的卷积核类似 Sobel，提取边缘、颜色
- **中层卷积**：组合边缘形成纹理、形状
- **深层卷积**：识别高级概念（人体、球拍）

### 2. 1×1 卷积（通道压缩）

```
输入 (80×80×256)
    ↓ Conv 1×1, 256→128
输出 (80×80×128)     ← 空间尺寸不变，通道数减半
```

**作用**：不改变空间尺寸，只调整通道数。相当于对每个像素位置做一次全连接变换。

### 3. 深度可分离卷积

将标准卷积分解为**逐通道卷积 + 逐点卷积**，减少计算量：

| 类型 | 计算量 |
|------|--------|
| 标准 3×3 卷积 (128→256) | 128 × 256 × 3 × 3 = 294,912 |
| 深度可分离卷积 | 128 × 3 × 3 + 128 × 256 = 33,920 |
| **节省** | **约 8.7 倍** |

## 卷积 vs 全连接——为什么卷积更适合图像

| 对比维度 | 全连接层 | 卷积层 |
|----------|----------|--------|
| 参数量 | 巨大（百万级） | 很少（几百~几千） |
| 平移不变性 | 无 | 有（同一个核扫全图） |
| 局部性 | 无（看全局） | 有（只看 3×3 区域） |
| 输入尺寸 | 固定 | 任意（核大小固定） |
| 适用场景 | 表格数据 | 图像、语音、视频 |
